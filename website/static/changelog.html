v1.11.0

  • Vale Server is now running the lastest version of Vale (v2.6.8)

v1.10.0

  • Vale Server is now available on all major Linux distributions through its new AppImage.
  • Updated dock icons on all platforms, and updated the tray icon on Windows for better contrast when using light-themed colors.
  • The VS Code, Atom, and Sublime Text clients now have a 'View Report' command that integrates with the new 'Summary' dashboard page.
  • Fixed an issue that could cause the dashboard to fail to create a new vocab entry.

v1.9.0

  • Vale Server's 'Studio' page is now publicly available at vale-studio.errata.ai.
  • Vale Server now has a new 'Summary' page, allowing users to create reports about local directories according to custom grading schemes.
  • Vale Server now supports three linting modes: Server (Vale Server only), Compatibility (Vale Server + Vale), Command Line (Vale only). The option has moved from the preferences window to the context menu.

v1.8.0

  • Vale Server now supports running LanguageTool as a background process, greatly improving on-the-fly linting performance.
  • Vale Server's action-processing engine has been completely rewritten, incorporating many performance improvements.
  • New versions of the Atom, Sublime Text, and VS Code clients are now available.

v1.7.0

  • Vale Server now supports specifying a "limit" how often a specific rule is reported.
  • Vale Server now supports a new extension point, "sequence," allowing for more grammar-focused rules.
  • Vale Server will now request permission (when not already available) to install an update on Windows instead of showing an error message.
  • Resolved an issue that caused some valid license keys to be denied upon initial validation.

v1.6.0

  • Windows 10 dark mode is now supported, giving the application a more native look when the mode is active.
  • Vocabulary files now accept regular expressions.
  • Only one instance of Vale Server may be active now, fixing an issue where multiple instances could be inadvertently started.
  • Multiple ini-related syntax highlighting improvements and fixes.
  • Vale Studio now correctly ignores code-related scopes.
